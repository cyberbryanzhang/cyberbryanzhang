### ☁️ WaterWatch Cloud Monitoring Service Case Study *(In Progress)*
A cloud computing case study for a fictional **WaterWatch** water monitoring service. The project studies how a finished cloud-based monitoring application could support small towns, farms, parks, and environmental groups without requiring them to build or manage their own backend infrastructure.

**Project idea**
- Uses sensors to collect water level, temperature, pH, and clarity data
- Sends sensor readings to the cloud for storage, dashboards, and reporting
- Provides a web/app dashboard where customers can view current water conditions
- Sends alerts when water readings change significantly, such as fast-rising river levels or low pond levels
- Evaluates why the service fits **Software as a Service (SaaS)** better than `IaaS` or `PaaS`

**What I’m practicing**
- Cloud service model analysis: `IaaS` vs `PaaS` vs `SaaS`
- Explaining cloud architecture from the customer’s point of view
- Connecting IoT-style sensor data to cloud dashboards and alerting
- Thinking through security and reliability needs such as accounts, access control, backups, updates, and provider responsibility
- Writing technical case studies in a clear, non-technical-user-friendly way

**Keywords:** `Cloud Computing` · `SaaS` · `IoT Sensors` · `Monitoring Dashboard` · `Alerting` · `Access Control` · `Reliability`
